# # CS 300 Portfolio Reflection

## What was the problem you were solving in the projects for this course?

The main problem I was solving in this course was how to organize and search course information in a way that is efficient and easy for users to use. The advising team needed a program that could load course data, show all courses in alphanumeric order, and find information about a specific course and its prerequisites. To do this, I had to compare different data structures and decide which one would work the best for the requirements.

## How did you approach the problem? Consider why data structures are important to understand.

I started by looking at the requirements and thinking about what operations the program would perform the most. Then I compared vectors, hash tables, and binary search trees. I analyzed the runtime and memory usage for each structure to understand their strengths and weaknesses.

Data structures are important because they affect how fast a program can find, store, and organize data. A program may still work with the wrong data structure, but it can become slower as more data is added. Understanding data structures helps a programmer make better decisions when designing software.

## How did you overcome any roadblocks you encountered while going through the activities or project?

One roadblock I had was understanding the different runtime complexities and knowing when one data structure would perform better than another. At first it was confusing to compare average case and worst case performance. I solved this by reviewing the course materials, practicing Big O analysis, and testing different approaches.

Another challenge was making sure the program could correctly read the course file and handle invalid data. I worked through this by adding checks for missing information and making sure the program could continue working even when errors were found.

## How has your work on this project expanded your approach to designing software and developing programs?

This project helped me think more about planning before writing code. Before this course I would often focus only on getting a program to run. Now I spend more time thinking about how data will be stored and accessed. I learned that choosing the right structure early can make the program easier to build and maintain later.

I also learned that software design is not only about writing code. It is also about analyzing requirements, comparing solutions, and selecting the best approach for the problem.

## How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

My programs are more organized now than when I started the course. I try to separate tasks into different functions so each part of the program has a clear purpose. This makes the code easier to read and easier to update if requirements change later.

I also learned the importance of meaningful variable names, comments, and error checking. These things may seem small, but they make the program much easier for another person to understand. Because of this project, I think more about how future users or developers may work with my code. It have helped me write programs that are more maintainable and adaptable than before.
